# Problem Solving

> Giải quyết các vấn đề phức tạp một cách có hệ thống thông qua phân tích đa chiều và khung tư duy có cấu trúc.

## Kỹ Năng Này Làm Gì

Skill `ckm:problem-solving` cung cấp các khung tư duy và phương pháp luận để phân tích, định nghĩa và giải quyết vấn đề phức tạp. Từ root cause analysis đến creative problem solving, phù hợp với cả thách thức kỹ thuật và kinh doanh.

## Bắt Đầu Nhanh

```
/ckm:problem-solving
```

Hoặc mô tả vấn đề:

```
/ckm:problem-solving Tỷ lệ churn tăng 15% trong 2 tháng — tìm nguyên nhân và giải pháp
```

## Các Khung Tư Duy

### 5 Whys — Tìm Nguyên Nhân Gốc Rễ
```
Vấn đề: Churn tăng
Tại sao 1: Người dùng không quay lại sau tuần đầu
Tại sao 2: Họ không thấy giá trị trong onboarding
Tại sao 3: Onboarding quá dài và phức tạp
Tại sao 4: Chúng ta chưa có persona-specific onboarding
Tại sao 5: Thiếu dữ liệu để segment người dùng khi đăng ký
→ Giải pháp gốc rễ: Thu thập use case khi signup
```

### MECE Framework — Phân Tích Toàn Diện
Mutually Exclusive, Collectively Exhaustive — đảm bảo phân tích không bỏ sót và không trùng lặp.

### First Principles — Tư Duy Từ Nền Tảng
Bóc tách giả định, xây dựng lại từ sự thật cơ bản thay vì tư duy tương tự.

## Tính Năng Chính

- **Root cause analysis**: 5 Whys, Fishbone diagram, Fault tree analysis
- **Problem framing**: Định nghĩa vấn đề đúng trước khi giải quyết
- **Solution generation**: Brainstorm có cấu trúc với đánh giá đánh đổi
- **Decision matrix**: So sánh giải pháp theo nhiều tiêu chí
- **Implementation roadmap**: Chuyển giải pháp thành kế hoạch hành động

## Ví Dụ Sử Dụng

**Phân tích vấn đề kinh doanh:**
```
/ckm:problem-solving CAC tăng 40% trong Q3 — phân tích nguyên nhân theo từng kênh
```

**Quyết định build vs buy:**
```
/ckm:problem-solving Đánh giá: tự xây dựng analytics pipeline hay dùng Mixpanel/Amplitude?
```

**Giải quyết mâu thuẫn team:**
```
/ckm:problem-solving Sales và Product bất đồng về roadmap priority — framework để align
```
