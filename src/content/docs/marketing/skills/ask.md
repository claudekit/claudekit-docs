# `ckm:ask`

> Get direct answers to technical, strategic, and architectural marketing questions.

## What This Skill Does

The Ask skill is the general-purpose Q&A interface for the marketing toolkit. It handles questions that don't fit neatly into a specific workflow — technical clarifications, strategy guidance, tool selection, and architectural decisions about your marketing stack.

Unlike brainstorming (which is exploratory and iterative), Ask is optimized for direct, concise answers. It reads project context, consults relevant documentation, and delivers a clear response without unnecessary back-and-forth.

The skill is particularly useful for onboarding new team members, resolving ambiguous implementation questions, and getting second opinions on marketing architecture decisions before committing to an approach.

## Quick Start

```
/ckm:ask How should I structure UTM parameters for a multi-touch attribution model?
```

Or conversationally:
```
Ask: What's the best way to segment email lists for a B2B SaaS audience?
```

## Key Features

- Context-aware answers using project docs and marketing stack config
- Technical depth calibrated to the question complexity
- Cites relevant references, docs, and best practices
- Covers: strategy, tooling, APIs, workflows, architecture, best practices
- No filler — concise, actionable responses

## Usage Examples

**Tool selection question**:
```
/ckm:ask Should we use HubSpot or Klaviyo for our email automation given we're B2B with 10k contacts?
```

**Technical clarification**:
```
/ckm:ask What's the difference between first-touch and last-touch attribution, and which should we use?
```

**Architecture decision**:
```
/ckm:ask We're building a content hub — should we store content in a headless CMS or flat markdown files?
```
